Login
Setar cookies, com a sessão, para manter ativa no front-end enquanto o servidor permitir
Ao fazer login retornar página com contexto, dependendo do tipo de usuário
Precisa incluir o cpf na tabela de sessão para poder definir qual a permissão do usuário que solicitou a sessão
Botão de sair, para excluir o cookie e deletar a sessão no servidor
Exibir a seção de cadastro e de login somente no hover sobre os botões do cabeçalho.
Ampulheta nas requisições ao servidor
O superusuario lista todos os usuários ao fazer login
Selecionar o usuário e obter todas as informações
O superusuário pode alterar o nível de permissão de outro usuário (nunca ao nível máximo, permitido somente ao supersusário)
O superusuario pode fazer refresh das sessões, excluindo as antigas
O superusuario pode excluir um usuário
O superusuário pode resetar a senha de outro usuário
O usuário gerente pode criar uma eleição, definindo candidatos e eleitores (que estejam cadastrados)
Uma eleição deve criar o bloco gênese da blockchain, atribuindo um saldo de 1 para cada eleitor
Usuários devem poder trocar a senha. Usuários podem esquecer a senha e receber por email um link para alterar a senha esse link deve conter uma sessão para acessar a alteração de senha do usuário (UPDATE modifiedAt e senhaa partir do cpf ) Usuários podem receber uma lista das eleições as quais estão cadastrados. Usuários podem votar em candidatos da eleição
